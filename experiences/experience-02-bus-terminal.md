# 차량·교통 단말기 개발

## 기간

2020.07 ~ 2023.08

**C++, Qt, QML, Linux, Kotlin, Android, Python**

---

## 한 줄 소개

운전자·승객 안내, 결제 연동, 운행 데이터 처리 단말기를 개발하며, 차량 디바이스 연동, UI 구조 개선과 제품화 지원을 담당했습니다.

## 주요 기여

### 안내·결제 단말기 기능 개발

- 운전자 단말기, 승객안내기, 결제 연동 단말기 UI 개발
- Qt C++ 기반 UI를 QML·C++ 기반 HMI로 전환하고, 상태·바인딩 기반 화면 갱신 구조 적용
- QML의 하드웨어 가속 렌더링 특성을 활용해 영상 재생 품질 개선
- 노선·정류장 정보 표시, 안내방송, 콘텐츠·외부 정보 연동 기능 구현
- 승객안내기에서 운행·정류장 상태에 따라 안내 정보와 영상 콘텐츠가 갱신되는 화면 흐름 구현
- 청각장애인 승객을 위한 수어 안내 기능 개발
- LTE, Wi-Fi, USB, RS-232, I2C·GPIO 기반 차량 디바이스 연동
- 파일 관리, 요금 처리, 프로세스 간 통신 등 단말기 기능 개발

### Android/Kotlin 기반 단말기 앱 개발

- Android/Kotlin 기반 운행 데이터 단말기 UI 개발 및 화면 전환 흐름 구현
- Activity·Service 기반 앱 구조를 구성하고, 백그라운드 작업과 화면 갱신 흐름 분리
- 서비스 실행 상태와 외부 앱 요청에 따라 다른 화면을 표시하는 단말기 앱 흐름 구현
- 브로드캐스트 기반 상태 전달과 설정 공유를 활용해 Activity·Service 간 데이터 연동
- 단말기 입출력 데이터와 Android 앱 화면을 연동해 운행 데이터 확인·백업 기능 구현

### 데이터 처리 및 개발 도구

- 노선 파일의 정류장 ID 배열과 정류장 상세 정보를 ID 기반 HashMap으로 조합해 UI용 단일 노선 데이터 생성 구조 설계
- 정류장 정보를 반복 탐색하던 조회 방식을 인덱스 조회로 전환해 데이터 조합 복잡도를 O(n²)에서 O(n)으로 개선
- Python·PySide6 기반 Windows 파일 검증 도구 개발
- 제품 검증용 테스트 프로그램 개발 및 제품화 과정 지원

### 구조 및 안정성 개선

- 데이터 처리와 UI 작업을 분리해 화면 지연 문제 개선
- Android 단말기 앱 개발 과정에서 UI 스레드와 작업 스레드의 책임을 분리해 화면 갱신 안정성 개선
- 명령 요청마다 독립 QThread에서 작업을 수행하고, 새 명령 처리 전 이전 작업 스레드와 객체를 정리하도록 실행 흐름 개선
- 기능을 모듈화하고 MVC를 적용해 유지보수 구조 개선
- 교통약자 정보 접근성을 위한 안내 기능을 구현하고, 관련 기술을 등록특허로 확장

### 기타 임베디드 프로젝트 경험

- 출입·설치 관리용 단말기 UI와 네트워크 설정 기능 개발
- IoT 게이트웨이 단말기 UI와 센서·서버 연동 기능 개발
- OpenCV 기반 영상 처리와 임베디드 환경에서의 운전자 보조 기능 프로토타입 개발 참여
