# 탐색 모듈 개발

## 기간

2023.12 ~ 현재

## 역할 및 기술 스택

- C++ 기반 탐색 기능 모듈 개발
- Kotlin 기반 모듈 전환을 위한 초기 구조 설계 및 핵심 기능 구현
- 개발 과제 분배, 구현 가이드, 주요 코드 리뷰
- 유관 부서와 기능 사양·탐색 동작 기준·연동 방식 협의

**C++, Kotlin, Android, Coroutine, StateFlow**

---

## 한 줄 소개

차량 탐색 기능의 C++ 기반 모듈을 개발하고, Android 기반 외부 연동 대응을 위한 Kotlin 전환 과정에서 초기 아키텍처와 핵심 탐색 기능을 설계·구현했습니다.

## 주요 기여

### C++ 기반 탐색 기능 개발

- 다중 경로 탐색 결과를 바탕으로 경로 선택과 안내 시작 기능 구현
- 현재 위치, 기존 경로, 경유지·목적지, 경로 옵션 상태를 기준으로 재탐색 요청 조건과 후처리 흐름 구현
- 목적지·경유지 추가, 삭제, 순서 변경 등 경로 편집과 탐색 요청 연동
- 실행 환경·유형별 경로 옵션 설정 및 변경에 따른 탐색 요청 처리
- 탐색 화면·정보 패널·이벤트에 따른 상태 전환 로직 구현
- 경로·목적지·경유지 표시를 위한 지도 연동 데이터 처리
- 전처리·후처리 흐름이 다른 탐색 유형을 기본 탐색 클래스와 하위 클래스로 분리해 기능 확장 구조 설계

### Kotlin 기반 모듈 전환

- Android 기반 외부 연동 대응이 필요해지면서 Controller 계층의 Kotlin 전환 구조 설계
- 기존 MVC 구조에서 Controller 비대화와 실행 환경별 조건 분기가 늘어나는 문제를 분석하고, 공통 탐색 흐름과 환경별 사양을 상속 구조로 분리
- 기존 동기 탐색 호출을 Coroutine 기반 비동기 호출 구조로 래핑
- 탐색 모듈 상태와 HMI Controller 상태를 분리하고, 모듈 상태를 화면 표현용 상태로 변환해 StateFlow로 전달
- 탐색 응답, 내부 데이터 모델, UI 표시 상태를 분리해 다른 탐색 모듈이 인입되어도 HMI 영향 범위를 줄일 수 있는 구조 설계
- 경로탐색·재탐색, 비교경로, 시간·거리 업데이트, 경로선 Polyline, 핀 표출, 지도 Fit 영역 계산 구현

## 구조 개선 및 문제 해결

### 탐색 상태와 UI 상태의 책임 분리

탐색 응답 데이터와 UI 계층이 동일한 데이터를 직접 공유하면 데이터 소유권과 변경 시점이 불명확해지고, 추후 다른 탐색 모듈 연동 시 HMI 영향 범위가 커질 수 있다고 판단했습니다.

- 탐색 응답을 기준 데이터로 관리하고, UI 표현용 상태를 별도 모델로 구성
- 탐색 모듈 상태를 HMI Controller 상태로 변환하고, StateFlow 기반 상태 변경에 따라 UI가 갱신되는 구조 구현

이를 통해 데이터 책임을 명확히 하고 기능 확장과 상태 관리가 쉬운 구조를 만들었습니다.

### Kotlin 전환 구조 설계 회고

초기 전환 과정에서는 Controller와 탐색 구현 간 결합도가 높아질 수 있는 구조적 한계를 확인했습니다. 이후 확장 관점에서 Controller가 필요한 계약을 인터페이스로 정의하고, 각 탐색 구현이 해당 계약을 구현하는 방식이 더 적절하다고 판단했습니다.

- HMI Controller는 특정 탐색 구현체보다 필요한 비즈니스 계약에 의존하는 구조가 더 적합하다고 회고
- 신규 탐색 구현 추가 시 Controller 변경을 줄이기 위해 어댑터·인터페이스 경계를 명확히 두는 방향으로 개선 필요성 도출

### 경로 옵션 관리 구조 개선

실행 환경·유형별로 분산되어 있던 경로 옵션 관리 방식을 통합했습니다.

- 옵션을 역할별로 분리하고 상위 관리 구조에서 통합 관리
- 화면 표시용 옵션과 실제 탐색 요청용 옵션의 책임 분리

이를 통해 신규 옵션 추가와 동작 기준 변경에 유연하게 대응할 수 있도록 개선했습니다.

### 화면 전환 시 UI 깜빡임 해결

상태 반영 시점과 화면 갱신 순서가 맞지 않아 화면 전환 시 UI 패널이 깜빡이는 문제를 해결했습니다.

- UI 갱신 흐름과 이벤트 처리 순서를 분석
- 화면 전환 전 필요한 상태가 반영되도록 처리 순서 조정

이를 통해 화면 전환 안정성을 개선했습니다.
