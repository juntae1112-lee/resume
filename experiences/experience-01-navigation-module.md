# 탐색 모듈 개발

## 기간

2023.12 ~ 현재

## 역할 및 기술 스택

- C++ 기반 탐색 기능 모듈 개발
- Kotlin 기반 모듈 전환을 위한 초기 구조 설계 및 핵심 기능 구현
- 개발 과제 분배, 구현 가이드, 주요 코드 리뷰
- 유관 부서와 기능 사양·탐색 정책·연동 방식 협의

**C++, Kotlin, Android, Coroutine, StateFlow**

---

## 한 줄 소개

차량 탐색 기능의 C++ 기반 모듈을 개발하고, Kotlin 전환을 위한 초기 아키텍처와 핵심 기능을 설계·구현했습니다.

## 주요 기여

### C++ 기반 탐색 기능 개발

- 다중 경로 탐색 결과를 바탕으로 경로 선택과 안내 시작 기능 구현
- 현재 경로 기반·주기적·사용자 요청 재탐색을 유형별 정책에 따라 처리
- 목적지·경유지 추가, 삭제, 순서 변경 등 경로 편집과 탐색 요청 연동
- 지역·플랫폼·유형별 경로 옵션 설정 및 변경에 따른 탐색 요청 처리
- 탐색 화면·정보 패널·이벤트에 따른 상태 전환 로직 구현
- 경로·목적지·경유지 표시를 위한 지도 연동 데이터 처리

### Kotlin 기반 모듈 전환

- 기존 UI 구조를 유지하며 Controller 계층을 Kotlin 기반으로 전환
- 동기 API를 Coroutine 기반 비동기 호출 방식으로 래핑
- API 응답, 내부 데이터 모델, UI 상태를 분리한 계층 구조 설계
- StateFlow 기반 상태 전달 구조 및 Polyline·거리·지도 표시 영역 데이터 변환 구현

## 구조 개선 및 문제 해결

### 탐색 상태와 UI 상태의 책임 분리

탐색 응답 데이터와 UI 계층이 동일한 데이터를 직접 공유하면 데이터 소유권과 변경 시점이 불명확해질 수 있다고 판단했습니다.

- 탐색 응답을 기준 데이터로 관리하고, UI 표현용 상태를 별도 모델로 구성
- 상태 변경에 따라 UI가 갱신되는 구조 구현

이를 통해 데이터 책임을 명확히 하고 기능 확장과 상태 관리가 쉬운 구조를 만들었습니다.

### 경로 옵션 관리 구조 개선

지역·플랫폼·유형별로 분산되어 있던 경로 옵션 관리 방식을 통합했습니다.

- 옵션을 역할별로 분리하고 상위 관리 구조에서 통합 관리
- 화면 표시용 옵션과 실제 탐색 요청용 옵션의 책임 분리

이를 통해 신규 옵션 추가와 정책 변경에 유연하게 대응할 수 있도록 개선했습니다.

### 화면 전환 시 UI 깜빡임 해결

상태 반영 시점과 화면 갱신 순서가 맞지 않아 화면 전환 시 UI 패널이 깜빡이는 문제를 해결했습니다.

- UI 갱신 흐름과 이벤트 처리 순서를 분석
- 화면 전환 전 필요한 상태가 반영되도록 처리 순서 조정

이를 통해 화면 전환 안정성을 개선했습니다.
